# 🎯 智能场景检测抽帧

## 💡 你的问题非常关键！

**不同视频需要不同的抽帧策略**

## 📊 视频类型分析

### 静态视频（变化少）
- **口播视频**：人物不动，只有嘴在动
- **访谈视频**：镜头固定，场景不变
- **PPT讲解**：画面切换慢

**问题**：1秒1帧太浪费，10帧都差不多

### 动态视频（变化多）
- **舞蹈视频**：动作快速变化
- **运动视频**：镜头快速切换
- **剪辑视频**：场景频繁变化

**问题**：1秒1帧可能漏掉关键动作

## 🔧 智能解决方案

### 场景检测算法

使用ffmpeg的**场景检测滤镜**：
```
select='gt(scene,0.3)'
```

**工作原理**：
- 分析相邻帧的差异
- 差异 > 30% → 提取这一帧
- 差异 < 30% → 跳过

### 自适应策略

```
1. 尝试场景检测
   ↓
2. 检测到的帧数够吗？
   ├─ 够（≥5帧）→ 使用场景检测结果 ✅
   └─ 不够（<5帧）→ 补充均匀采样 🔄
```

## 🎬 实际效果

### 场景检测参数

```python
scene_threshold = 0.3  # 30%变化阈值
```

**阈值说明**：
- **0.1**：非常敏感，轻微变化就提取
- **0.3**：适中，明显变化才提取（推荐）
- **0.5**：不敏感，只在大变化时提取

### 提取逻辑

```
视频播放中...
├─ 画面变化 < 30% → 跳过
├─ 画面变化 ≥ 30% → 提取 ✅
└─ 重复检测...
```

## 💡 智能补充机制

如果场景检测提取太少（< 5帧）：
- 说明视频太静态
- 自动切换为均匀采样
- 确保至少有10帧

**示例**：
```
场景检测: 只提取到3帧（视频太静态）
↓
自动补充: 均匀采样10帧
↓
最终: 10帧均匀分布
```
