## ❌ 之前的问题

你的7分钟视频只提取了10帧：
- 视频时长：479秒（7分59秒）

## ✅ 现在的解决方案

**根据视频时长自适应调整帧数**

### 帧数策略

| 视频时长 | 提取帧数 | 覆盖间隔 | 效果 |
|---------|---------|---------|------|
| ≤ 30秒 | 10帧 | 每3秒 | ✅ 完整覆盖 |
| 31-60秒 | 15帧 | 每4秒 | ✅ 完整覆盖 |
| 1-3分钟 | 20帧 | 每9秒 | ✅ 完整覆盖 |
| 3-5分钟 | 25帧 | 每12秒 | ✅ 完整覆盖 |
| **> 5分钟** | **30帧** | **每16秒** | **✅ 完整覆盖** |

### 你的7分钟视频

**之前**：
- 10帧 ÷ 479秒 = 每48秒1帧
- 只覆盖前480秒（8分钟）的10个点
- **后面内容完全看不到** ❌

**现在**：
- 30帧 ÷ 479秒 = 每16秒1帧
- 完整覆盖整个7分钟
- **AI能看到完整视频** ✅

## 🎬 智能场景检测 + 自适应帧数

### 工作流程

```
1. 获取视频时长（479秒）
   ↓
2. 自动计算帧数（30帧）
   ↓
3. 场景检测提取关键帧
   ├─ 在场景变化时提取
   ├─ 最多提取30帧
   └─ 均匀分布在整个视频
   ↓
4. 发送给AI分析
   ↓
5. AI看到完整视频内容 ✅
```

### 场景检测的优势

即使是30帧，也会优先提取：
- 场景切换的时刻
- 动作变化的时刻
- 关键画面

而不是机械地每16秒提取一次。

## 💡 为什么30帧是上限？

### API限制考虑

1. **请求体大小**
   - 30帧 × 50KB = 1.5MB
   - 大多数API支持
   - 稳定可靠

2. **Token消耗**
   - 每张图片消耗约85 tokens
   - 30帧 = 2550 tokens
   - 加上文本 ≈ 3000 tokens
   - 成本可控

3. **处理时间**
   - 提取30帧：约10秒
   - AI分析：约60秒
   - 总计：约70秒
   - 用户可接受

### 如果需要更多帧？

可以修改配置：

```python
# 在 ai_config.yaml 中添加
AI:
  MAX_FRAMES: 50  # 增加到50帧
```

但要注意：
- 数据量更大
- 成本更高
- 可能超时
